# Self-hosted

The [hypergraph_memory](https://github.com/RunStack-AI/hypergraph_memory) repository contains the full open-source server. Running it locally requires no account, no API key, and no network — memory is stored in a local JSON file.

<Info>
The self-hosted server exposes `memory_*` tools (e.g. `memory_store`, `memory_recall`). The hosted service at `api.hypermemory.io` exposes `hm_*` tools (e.g. `hm_store`, `hm_recall`) with additional features like hybrid search, enrichment, timeline, and file storage.
</Info>

## Clone and install

```bash
git clone https://github.com/RunStack-AI/hypergraph_memory.git
cd hypergraph_memory
pip install -e ".[mcp]"
```

Core dependencies: `pydantic>=2.0`, `portalocker>=2.0`, plus the `mcp` extra for `mcp>=1.0`.

## Run: stdio (default)

From the repository root:

```bash
python src/server.py
```

This starts an MCP server on **stdio** via `mcp.run()`. Desktop MCP clients (Claude Desktop, Claude Code) spawn this as a subprocess.

### Claude Desktop config (self-hosted)

```json
{
  "mcpServers": {
    "hypergraph-memory": {
      "command": "python3",
      "args": ["/absolute/path/to/hypergraph_memory/src/server.py"],
      "env": {
        "HYPERGRAPH_MEMORY_FILE": "/absolute/path/to/memory.json"
      }
    }
  }
}
```

## Run: HTTP

The HTTP wrapper lives at `deploy/mcp-server/mcp_http_server.py`. It imports the same FastMCP app and serves it over Streamable HTTP:

```bash
python deploy/mcp-server/mcp_http_server.py
```

Or with uvicorn:

```bash
uvicorn deploy.mcp-server.mcp_http_server:app --host 0.0.0.0 --port 8080
```

## Environment variables

| Variable | Default | Description |
|---|---|---|
| `HYPERGRAPH_MEMORY_FILE` | `./memory.json` | Path to the JSON persistence file |
| `AUTO_LOAD_WRITABLE` | `false` | Load linked hypergraphs as writable on startup |
| `ALLOW_SET_FILE` | `true` | Register the `memory_set_file` tool to switch active files at runtime |
| `MCP_HOST` | `0.0.0.0` | Bind host (HTTP mode only) |
| `MCP_PORT` | `8080` | Bind port (HTTP mode only) |
| `LOGGER_NAME` | module name | Logger name for file logging |
| `LOGGER_PATH` | none | Path for log file output |

## Test agent

`agent.py` in the repo root is a PydanticAI chat agent that spawns the MCP server as a child process:

```bash
export OPENROUTER_API_KEY=...
export HYPERGRAPH_MEMORY_FILE=./memory.json
python agent.py
```

It uses `MCPServerStdio("python", ["./src/server.py"], env=...)` and supports `--model`, `--agent`, and `--memory-file` flags.

## No authentication

The self-hosted server has no built-in auth. If you expose it over a network, put a reverse proxy with authentication in front of it. The `service/` directory in the repo contains the gateway and middleware used by the hosted product, but it is not required for local use.

## OpenAPI spec

The repo includes `docs/api/openapi.yaml` describing the hosted gateway's REST endpoints (`/v1/mcp`, `/keys`, `/usage`, etc.). This spec is for reference — the local server does not implement the gateway routes.
