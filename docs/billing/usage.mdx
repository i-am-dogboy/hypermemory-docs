# Usage

## What counts as a query

Every `tools/call` request to the hosted MCP endpoint counts as one query, regardless of which tool is called. `tools/list` does not count.

REST API calls to `/api/v1/memory/*` endpoints also count as queries.

## Viewing usage

Go to [app.hypermemory.io/usage](https://app.hypermemory.io/usage) to see:

- Current period query count vs plan limit
- Daily usage breakdown

## Monthly query limits

| Plan | Monthly queries |
|---|---|
| Free | 2,000 |
| Basic | 10,000 |
| Pro | 200,000 |
| Business | 500,000 |
| Enterprise | Unlimited |
| RunStack | Unlimited |

## Rate limits

Rate limits are enforced per tenant, per minute:

| Plan | Write | Read | Ingest |
|---|---|---|---|
| Free | 10 | 30 | 3 |
| Basic | 30 | 60 | 5 |
| Pro | 100 | 200 | 20 |
| Business | 200 | 400 | 40 |
| Enterprise | 300 | 600 | 60 |
| RunStack | 300 | 600 | 60 |

Exceeding the rate limit returns HTTP 429.

**Write** tools: `hm_store`, `hm_update`, `hm_forget`, `hm_upload_file`, `hm_timeline_write`

**Read** tools: `hm_recall`, `hm_find_related`, `hm_get_overview`, `hm_list_files`, `hm_timeline`

**Ingest** tool: `hm_ingest`

## REST API usage endpoints

| Endpoint | Method | Description |
|---|---|---|
| `/v1/usage` | GET | Current period usage |
| `/v1/usage/history` | GET | Daily usage snapshots |

Authenticate with a Bearer API key or OAuth token.

## Self-hosted

The open-source server has no usage tracking or rate limiting.
