# hm_find_related

Find nodes connected to a given node by traversing hyperedge relationships. Supports multiple traversal strategies for different use cases.

## Parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `key` | `str` | **required** | Starting node key |

## Traversal strategies

The hosted service supports three traversal strategies:

| Strategy | Description |
|---|---|
| **Chain** | Follow edges step-by-step from the starting node |
| **Decay** | Weight connections by distance — closer nodes score higher |
| **Type-boosted** | Prioritize specific node types in traversal |

## Response

Returns the subgraph reachable from the starting node — connected nodes with their relationships and metadata.

## Example

```json
{
  "key": "person_alice"
}
```

## CLI equivalent

```bash
hm find person_alice --depth 2
```

## REST endpoint

```
POST /api/v1/memory/find-related
```

## Self-hosted equivalent

The open-source server's `memory_find_related` accepts `key`, `relationship_pattern` (regex filter on edge labels), `max_depth` (1–5, default 2), and `max_nodes` (1–100, default 30).
