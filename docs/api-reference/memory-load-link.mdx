# REST API Endpoints

For clients that don't use MCP, HyperMemory provides a REST API that maps directly to the MCP tools. All endpoints accept JSON bodies and require authentication.

## Authentication

Include one of:

- **API key:** `Authorization: Bearer hm_YOUR_KEY`
- **OAuth token:** `Authorization: Bearer <supabase_jwt>`

## Base URL

```
https://api.hypermemory.io/api/v1/memory
```

## Endpoints

### Store

```
POST /api/v1/memory/store
```

Create a node. Maps to [`hm_store`](/api-reference/memory-store).

```json
{
  "key": "fact_launch_date",
  "description": "Product launches on 2026-06-01",
  "node_type": "fact",
  "data": {"date": "2026-06-01"}
}
```

### Recall

```
POST /api/v1/memory/recall
```

Hybrid search. Maps to [`hm_recall`](/api-reference/memory-recall).

```json
{
  "query": "launch date"
}
```

### Update

```
POST /api/v1/memory/update
```

Update a node. Maps to [`hm_update`](/api-reference/memory-update).

```json
{
  "key": "fact_launch_date",
  "description": "Product launch postponed to 2026-07-01",
  "data": {"date": "2026-07-01", "status": "postponed"}
}
```

### Forget

```
POST /api/v1/memory/forget
```

Delete a node. Maps to [`hm_forget`](/api-reference/memory-forget).

```json
{
  "key": "fact_old_info",
  "cascade": true
}
```

### Ingest

```
POST /api/v1/memory/ingest
```

Decompose text into entities. Maps to [`hm_ingest`](/api-reference/memory-add-relationships).

```json
{
  "text": "Sarah joined as CTO in January...",
  "context": "Team update"
}
```

### Overview

```
GET /api/v1/memory/overview
```

Graph stats. Maps to [`hm_get_overview`](/api-reference/memory-get-overview).

### Add relationships

```
POST /api/v1/memory/relationships
```

Create edges between existing nodes. Accepts a list of relationship objects, each with `nodes` (list of 2+ keys) and `relationship` (label string).

```json
{
  "relationships": [
    {
      "nodes": ["person_alice", "project_phoenix"],
      "relationship": "Alice leads Project Phoenix"
    }
  ]
}
```

### Get relationships

```
GET /api/v1/memory/relationships/{key}
```

List all edges connected to a specific node.

### Find related

```
POST /api/v1/memory/find-related
```

Graph traversal. Maps to [`hm_find_related`](/api-reference/memory-find-related).

```json
{
  "key": "person_alice"
}
```

### Timeline

```
POST /api/v1/memory/timeline
```

Temporal recall. Maps to [`hm_timeline`](/api-reference/memory-export-subgraph).

### Timeline write

```
POST /api/v1/memory/timeline/write
```

Write a diary entry. Maps to [`hm_timeline_write`](/api-reference/memory-export-subgraph).

### Health

```
GET /api/v1/memory/health
```

Check server connectivity and status.

### Export

```
GET /api/v1/memory/export
```

Export the full graph as JSON.
