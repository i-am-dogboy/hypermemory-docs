# API Reference

The hosted HyperMemory service at `api.hypermemory.io` exposes 11 MCP tools over Streamable HTTP at `/mcp`. These tools are also accessible via the REST API and the `hm` CLI.

## MCP tools

### Core tools

| Tool | Description |
|---|---|
| [`hm_store`](/api-reference/memory-store) | Create a node; triggers background enrichment and entity detection |
| [`hm_update`](/api-reference/memory-update) | Update a node's fields; re-embeds on description change |
| [`hm_recall`](/api-reference/memory-recall) | Hybrid search (BM25 + vector + edge BM25 + regex) |
| [`hm_find_related`](/api-reference/memory-find-related) | Graph traversal from a starting node |
| [`hm_get_overview`](/api-reference/memory-get-overview) | Graph stats, top nodes, type distribution |
| [`hm_forget`](/api-reference/memory-forget) | Delete a node with optional cascade |
| [`hm_ingest`](/api-reference/memory-add-relationships) | Decompose dense text into graph entities in one LLM pass |

### File tools

| Tool | Description |
|---|---|
| [`hm_upload_file`](/api-reference/memory-get-relationships) | S3 upload + AI summary + graph node (Pro+ plans) |
| [`hm_list_files`](/api-reference/memory-get-relationships) | Query uploaded files by type or name |

### Timeline tools

| Tool | Description |
|---|---|
| [`hm_timeline`](/api-reference/memory-export-subgraph) | Temporal recall with query, period, date range, and node_key filters |
| [`hm_timeline_write`](/api-reference/memory-export-subgraph) | Write an explicit diary entry |

## REST API endpoints

All REST endpoints accept JSON bodies and require `Authorization: Bearer hm_YOUR_KEY` or a Supabase JWT.

| Method | Path | Maps to |
|---|---|---|
| POST | `/api/v1/memory/store` | hm_store |
| POST | `/api/v1/memory/recall` | hm_recall |
| POST | `/api/v1/memory/update` | hm_update |
| POST | `/api/v1/memory/forget` | hm_forget |
| POST | `/api/v1/memory/ingest` | hm_ingest |
| GET | `/api/v1/memory/overview` | hm_get_overview |
| POST | `/api/v1/memory/relationships` | hm_add_relationships |
| GET | `/api/v1/memory/relationships/{key}` | hm_get_relationships |
| POST | `/api/v1/memory/find-related` | hm_find_related |
| POST | `/api/v1/memory/timeline` | hm_timeline |
| POST | `/api/v1/memory/timeline/write` | hm_timeline_write |
| GET | `/api/v1/memory/health` | hm_health |
| GET | `/api/v1/memory/export` | hm_export_full |

## Self-hosted tools

The open-source [hypergraph_memory](https://github.com/RunStack-AI/hypergraph_memory) server exposes `memory_*` tools (e.g. `memory_store`, `memory_recall`) plus linked hypergraph management tools. See [Self-hosted](/connect/self-hosted) and [Linked hypergraphs](/concepts/linked-hypergraphs) for details.
