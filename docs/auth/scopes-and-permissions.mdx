# Scopes & permissions

## API key scopes

When you create an API key, you select a scope that determines the key's access level:

| Scope | Access | Use case |
|---|---|---|
| `owner` | Read + write | Personal keys, full agent access |
| `member` | Read-only | Organization keys, shared read access |

Default for personal keys: **`owner`**.

## OAuth scopes

The OAuth 2.1 flow advertises three scopes:

| Scope | Description |
|---|---|
| `memory:read` | Read nodes, edges, and graph state |
| `memory:write` | Create and update nodes and edges |
| `memory:admin` | Full access including administrative operations |

<Info>
Scope enforcement on the hosted service is primarily through **plan limits and rate limits**, not per-scope-string gating. All authenticated users have access to the tools their plan allows.
</Info>

## Plan-based enforcement

Access to certain features depends on your plan tier:

| Feature | Free | Basic | Pro | Business | Enterprise |
|---|---|---|---|---|---|
| File upload (`hm_upload_file`) | No | No | Yes | Yes | Yes |
| Multiple graphs | 1 | 1 | 4 | 20 | Unlimited |
| Write rate limit (per min) | 10 | 30 | 100 | 200 | 300 |
| Read rate limit (per min) | 30 | 60 | 200 | 400 | 600 |
| Ingest rate limit (per min) | 3 | 5 | 20 | 40 | 60 |

See [Plans](/billing/plans) for the complete tier comparison.

## Self-hosted

The open-source server does not enforce scopes or plan limits — all tools are available to any caller.
